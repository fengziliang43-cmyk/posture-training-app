# 体态训练 App V2 设计规格

**目标：** 在第一版已经能生成训练、记录趋势、上传体态照片和打包 APK 的基础上，第二版把 App 升级成更会解释、更会复盘、更容易坚持的私人训练系统。

**核心原则：** 训练决策仍由本地规则引擎负责；DeepSeek 只做解释、总结和保守建议，不直接决定动作、不读取体态照片、不替代医疗判断。

**架构延续：** OPPO APK 访问 Mac server；SQLite 保存结构化数据；体态照片保存在本地文件系统；DeepSeek API key 只保存在 Mac server 本地设置中，不写入代码、git、APK 或 Obsidian 记忆。

---

## 1. V2 产品定位

V1 解决的是：

```text
今天状态如何 -> 今天练什么 -> 训练和记录能不能跑通
```

V2 解决的是：

```text
为什么这么练 -> 这周发生了什么 -> 下周怎么更稳地继续
```

V2 不追求功能花哨，而是补足长期使用时最容易缺的三件事：

- 解释：让良知道今天为什么是完整、降级或恢复。
- 复盘：让良每周看到趋势，而不是只看某一天。
- 调整：当动作不合适时，能低成本换成替代动作。

## 2. V2 范围

V2 第一阶段做 5 个模块：

1. DeepSeek 设置和连接测试。
2. 今日训练解释。
3. 每周复盘。
4. 动作替换。
5. 体态照片对比增强和本地备份入口。

其中 DeepSeek 相关功能必须是可选的。未配置 API key 时，App 仍完整可用，只显示规则引擎的本地解释。

## 3. DeepSeek 定位

DeepSeek 可以负责：

- 把规则引擎的计划原因改写成更自然的教练口吻。
- 根据最近 7 天记录生成每周总结。
- 根据趋势提出保守建议，例如“这周先维持，不进阶”。
- 帮助解释某个动作在今天计划里的作用。

DeepSeek 不可以负责：

- 直接选择训练动作。
- 自由生成新的训练计划。
- 建议大重量、高冲击或激进进阶。
- 分析体态照片。
- 读取或保存 API key 之外的敏感凭证。
- 作医疗诊断或替代医生意见。

AI 输出必须经过本地安全边界包裹。任何 AI 文案都要带有保守约束：腰酸升高、红旗症状或明显不适时停止训练。

## 4. API Key 和隐私

API key 的规则：

- 只在设置页由良手动输入。
- 只提交给 Mac server。
- 只保存在本地 SQLite settings 表或后续本地安全存储中。
- 前端只显示“已配置/未配置”，不回显完整 key。
- 日志、错误、测试输出不得打印完整 key。
- APK 不内置 key。
- git、docs、Obsidian 记忆不得记录 key。

设置页应该支持：

- 输入或替换 DeepSeek API key。
- 清除 DeepSeek API key。
- 测试 DeepSeek 连接。
- 显示最近一次测试状态和时间。

## 5. 今日训练解释

今日页在生成计划后新增一个“今日解释”区域。

默认本地解释内容：

- 今日状态判断：完整、降级或恢复。
- 触发原因：腰酸、精神、睡眠、时间或周训练次数。
- 主要注意事项。

如果 DeepSeek 已配置并开启，可以生成更自然的解释：

- 语气像教练，短句，直接。
- 不超过 120 字。
- 不新增训练动作。
- 不修改组数、次数、休息时间。
- 不鼓励硬扛。

示例：

```text
今天精神一般，所以保留训练框架但降低总量。重点是稳定上背和核心，不追求疲劳。腰酸升高就停，宁可少做一点。
```

## 6. 每周复盘

记录页新增“本周复盘”卡片。

本地规则先计算：

- 本周训练完成次数。
- 本周平均腰酸。
- 本周最高腰酸。
- 睡眠平均值。
- 精神平均值。
- 训练后腰酸是否升高。
- 是否满足保守进阶条件。

无 DeepSeek 时显示结构化本地总结。

有 DeepSeek 时可以生成 3 段短文：

- 本周状态。
- 下周训练建议。
- 需要注意的风险。

AI 每周复盘不得直接说“下周加重量”或“强度拉满”。进阶建议必须引用本地规则判断，例如“如果下周腰酸仍稳定，可以考虑按规则小幅进阶”。

## 7. 动作替换

今日任务每个动作已有替代动作。V2 将“替代动作”从静态文字升级成按钮。

交互：

- 每个动作卡片显示“换成替代动作”。
- 点击后替换为动作库中的 replacementId 对应动作。
- 替换后保留同一训练计划 id，并记录替换日志。
- 支持撤回一次替换。

约束：

- 只能换成动作库里已有动作。
- 不能自由输入动作。
- 替换不能绕过恢复日或红旗症状限制。
- 替换后的动作也必须显示组数、次数/时间、休息、注意事项和停止条件。

## 8. 体态照片对比增强

V2 不做 AI 照片分析，只改善人工对比。

新增能力：

- 同角度时间轴。
- 最新照片 vs 上一次照片。
- 本月第一张 vs 最新照片。
- 每张照片可写简短备注。
- 对比页保留正面、侧面、背面分组。

照片仍只保存在手机和 Mac 本地，不上传第三方服务。

## 9. 本地备份

V2 增加数据安全入口。

第一阶段做：

- 导出本地备份 zip。
- 备份包含 SQLite 数据和上传照片。
- 备份文件由良手动保存。
- 设置页显示备份入口。

恢复备份可以作为 V2 后半段或 V3，不强行塞进第一阶段。

## 10. 页面变化

今日页：

- 今日任务下新增“今日解释”。
- 动作卡片新增“换成替代动作”。

记录页：

- 月历保持在顶部。
- 新增“本周复盘”卡片。
- DeepSeek 开启时可生成 AI 复盘。

体态页：

- 增强同角度对比。
- 增加照片备注。

设置页：

- DeepSeek API key 配置。
- DeepSeek 开关和测试连接。
- 本地备份导出。

## 11. 错误和降级

DeepSeek 失败时：

- 不影响生成训练计划。
- 不影响保存记录。
- 显示“AI 解释暂不可用，本地规则仍正常”。
- 自动回退到本地规则解释。

网络不可用时：

- 今日计划和离线打卡保持 V1 行为。
- AI 功能不进入离线队列，避免重复扣费和状态混乱。

API key 无效时：

- 设置页提示重新配置。
- 不在错误里显示完整 key。

## 12. 安全验收标准

V2 通过标准：

- 没有 API key 被写进代码、docs、git diff、日志或 APK。
- DeepSeek 关闭时 App 完整可用。
- DeepSeek 不能改变训练动作、训练类型、组数、次数或休息时间。
- 红旗症状仍优先阻断训练。
- 照片不发送给 DeepSeek。
- 动作替换只使用本地动作库。
- 本地测试、类型检查、构建和 APK 打包通过。

## 13. V2 不做什么

V2 第一阶段不做：

- AI 体态照片分析。
- AI 自由生成训练计划。
- 医疗诊断。
- 可穿戴设备接入。
- 自动热量计算。
- 公开账号、云同步、社交分享。
- 自动恢复备份。
